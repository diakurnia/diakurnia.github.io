Overview

A growing modular construction company — with flagship projects like the Wellington Temporary Ferry Terminal for Interislander — was losing valuable time on repetitive sales and admin tasks. Sales teams manually reviewed every inbound lead, proposal writing took days, and hot leads went cold due to slow response times.

This project delivers a 4-agent AI automation stack built on Claude + n8n + Supabase. Every agent drafts — every output requires human approval before reaching a client. The system handles the repetitive cognitive work; the team handles judgement and relationships.

The 4-Agent System
AgentNameWhat it doesStatus
Agent 1 Lead Scoring Scores inbound leads 0–100 against ICP within 2 minutes of form submission. Posts result to Telegram with recommended next action. Saves to Supabase. Live
Agent 2 Proposal Drafting Auto-triggered when a lead scores "hot". Drafts full client proposals using a RAG scope library (5 templates) and project history. Outputs structured draft to Telegram for review. Live
Agent 3 Post-Sales Ops Daily digest across 3 Airtable tables. Drafts client status updates, supplier POs, council RFI responses, and invoice follow-ups. All queued for approval. Planned
Agent 4 PM Daily Brief Morning digest of all active projects enriched with live Wellington weather data. Flags missed variation claims and milestone billing before they slip. Planned
How It Works — End to End
1
Lead submits Google Form Prospect fills out a lead intake form. Response auto-syncs to Google Sheets, triggering Agent 1 within 1 minute.
2
Agent 1 scores the lead Claude Sonnet 4.6 (temp: 0.1) evaluates the lead against ICP criteria — industry, project value, location, urgency — and outputs a score 0–100 with tier (hot/warm/cold) and recommended action. Result saved to Supabase.
3
Telegram notification to team The score, tier, and recommended next action are posted to the team's Telegram channel for immediate review. No lead goes unseen.
4
Hot leads trigger Agent 2 (fire-and-forget) If tier = hot, Agent 1 triggers Agent 2 as a sub-workflow without waiting. Agent 2 pulls 5 scope templates from an n8n Data Table (RAG pattern), aggregates them into a proposal context, and sends to Claude.
5
Proposal draft delivered to Telegram The full structured proposal draft — executive summary, scope, pricing range, timeline, next steps — lands in Telegram for PM review before any client contact.
Architecture Notes
Sub-workflow orchestration. Agent 2 runs as a separate n8n workflow triggered by Agent 1 with waitForSubWorkflow: false — keeping Agent 1 fast and non-blocking while Agent 2 handles the heavier proposal drafting asynchronously. Structured output uses schemaType: manual with JSON Schema constraints and an autoFix model at temp 0 to enforce reliable output format.
Human-in-the-loop guarantee. Every agent output goes to Telegram for review before reaching a client. No automated emails, no auto-sent proposals. The system drafts — the team decides.
Key Engineering Patterns
  • Structured output + autoFix at temp 0
  • RAG via n8n Data Table (scope library)
  • Fire-and-forget sub-workflow trigger
  • Aggregate → Set for multi-source context
  • nodeJson() for cross-node field access
  • Optional-chaining for input normalization
  • Fan-out → Merge (3–4 inputs) pattern
  • Real-time weather API integration
